# Security Policy

## Supported Versions

SecureDID v4 is currently under active development.

---

# Reporting Vulnerabilities

Please do not publicly disclose security vulnerabilities.

For sensitive vulnerabilities:
- Contact privately before opening public issues

For non-sensitive bugs:
- Open a GitHub issue

---

# Do Not Share

Never share:
- private keys
- wallet seed phrases
- Pinata JWTs
- `.env.local`
- production secrets

---

# Security Features

SecureDID v4 includes:
- DID-based verification
- Wallet authentication
- Governance approvals
- Revocation handling
- Access expiration (TTL)
- Encrypted VC storage
- Replay attack mitigation
- Identity spoofing prevention

---

# Disclaimer

This project is currently experimental and deployed on Base Sepolia testnet.
